# GritInk Digital Marketing Portfolio

A bold, multi-page portfolio website for a digital marketing company, built with TanStack Start, React 19, TypeScript, and Tailwind CSS v4. The site features a neon mint / sports-impact design direction, scroll-reveal animations, and a cinematic video reel on the homepage.

---

## Tech Stack

- **Framework:** [TanStack Start](https://tanstack.com/start) (full-stack React framework)
- **Bundler:** [Vite](https://vitejs.dev/) (configured via `@lovable.dev/vite-tanstack-config`)
- **UI:** React 19, TypeScript, Tailwind CSS v4, Radix UI primitives, shadcn/ui components
- **Routing:** `@tanstack/react-router` (file-based routing)
- **Fonts:** Bebas Neue (display), Barlow (body) via Fontsource
- **Package Manager:** [Bun](https://bun.sh/)

---

## Prerequisites

- [Bun](https://bun.sh/) installed locally (`bun --version` should print `1.x` or later)
- Node.js is **not required** if you use Bun, but the project is compatible with Node + `npm`/`pnpm` if you prefer

## Install

Clone the repo and install dependencies:

```bash
git clone <repository-url>
cd <project-folder>
bun install
```

---

## Run (Development)

Start the Vite dev server:

```bash
bun dev
```

Then open [http://localhost:8080](http://localhost:8080) in your browser.

> The dev server port is auto-detected by the Lovable sandbox config; it usually starts on port `8080`.

## Environment Variables

This project is designed to run without backend configuration out of the box. If you connect it to **Lovable Cloud** (Supabase) for features like contact form storage, auth, or a database, add the relevant variables to a `.env` file at the project root:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-anon-key
```

Server-only variables are read inside `createServerFn` handlers and should not be prefixed with `VITE_`:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_PUBLISHABLE_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

See the [Lovable Cloud docs](https://docs.lovable.dev/features/cloud) for details.
